## 功能描述
- 在“描述内容”区域的“技能内容”标签旁新增一个“同步”按钮。
- 点击后按顺序尝试拉取 `SKILL.md` 文本并填充到内容输入框：
  1. 直接请求：`gitUrl + '/SKILL.md'`
  2. GitHub 原始内容：`https://raw.githubusercontent.com/{owner}/{repo}/{branch}/SKILL.md`（分支默认 `main`，回退 `master`）
  3. GitLab 原始内容：`https://gitlab.com/{owner}/{repo}/-/raw/{branch}/SKILL.md`（默认 `main`，回退 `master`）
- 拉取失败时弹提示：“未获取到技能内容，请确认Git地址”。

## 前端改动（仅 AdminSkills.vue）
- UI：在“技能内容”标签右侧添加一个按钮（`同步`），显示加载状态（转圈/禁用），无后端改动。
- 状态与方法：
  - `isSyncing: boolean` 控制按钮状态
  - `syncContentFromRepo(): Promise<void>` 执行拉取与填充
  - 辅助：`buildCandidateUrls(gitUrl: string): string[]`、`fetchFirstOk(urls: string[]): Promise<string|null>`、`normalizeGitUrl(gitUrl)` 解析 owner/repo/branch
- 校验：
  - `git_url` 必须以 `http(s)://` 开头；为空或不合法直接提示
  - 内容非空才填充；保留已有内容可二次覆盖（直接替换）
- 错误处理：
  - 404/网络错误/CORS 均视为失败并提示
  - 超时时间（如 8s），超过则失败

## 代码要点（TypeScript 伪代码）
```ts
/**
 * 点击同步：尝试多种原始地址拉取 SKILL.md 文本
 */
async function syncContentFromRepo(): Promise<void> {
  if (!form.value.git_url || !/^https?:\/\//.test(form.value.git_url)) {
    alert('未获取到技能内容，请确认Git地址')
    return
  }
  try {
    isSyncing.value = true
    const urls = buildCandidateUrls(form.value.git_url)
    const text = await fetchFirstOk(urls)
    if (text && text.trim()) {
      form.value.content = text
      autoResizeInstallCommand(/* 复用或实现内容区域自适应 */)
    } else {
      alert('未获取到技能内容，请确认Git地址')
    }
  } finally {
    isSyncing.value = false
  }
}

/** 构造候选原始内容地址 */
function buildCandidateUrls(gitUrl: string): string[] {
  const list = [`${stripTrailingSlash(gitUrl)}/SKILL.md`]
  const gh = parseGithub(gitUrl) // { owner, repo, branch }
  if (gh) {
    list.push(`https://raw.githubusercontent.com/${gh.owner}/${gh.repo}/${gh.branch}/SKILL.md`)
    list.push(`https://raw.githubusercontent.com/${gh.owner}/${gh.repo}/master/SKILL.md`)
  }
  const gl = parseGitlab(gitUrl) // { owner, repo, branch }
  if (gl) {
    list.push(`https://gitlab.com/${gl.owner}/${gl.repo}/-/raw/${gl.branch}/SKILL.md`)
    list.push(`https://gitlab.com/${gl.owner}/${gl.repo}/-/raw/master/SKILL.md`)
  }
  return Array.from(new Set(list))
}

/** 逐个尝试请求，第一个 2xx 返回文本 */
async function fetchFirstOk(urls: string[]): Promise<string|null> {
  for (const u of urls) {
    try {
      const c = await fetchWithTimeout(u, { headers: { Accept: 'text/plain' } }, 8000)
      if (c.ok) return await c.text()
    } catch {}
  }
  return null
}
```

## 验证与兼容
- 浏览器跨域：GitHub/Raw 与 GitLab 原始内容通常允许 CORS；直接 `repo + /SKILL.md` 可能不通，故使用多候选。
- 仅填充文本，无安全风险；前端渲染已使用 `markdown-it + DOMPurify`。
- 无需后端改动与新依赖。
